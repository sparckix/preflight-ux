Brief Run Normalize Report

1. Review brief

This example uses the Preflight UX web app itself as the product surface. The review brief combines audience, stage, prototype URL, surface notes, selected personas, and risk focus areas.

{
  "artifact_type": "preflight_ux_review_brief",
  "run_id": "preflight-ux-web-ui-2026-05-12",
  "product": {
    "name": "Preflight UX web UI",
    "audience": "Product builders and design reviewers",
    "stage": "prototype",
    "surface_type": "prototype",
    "prototype_url": "https://preflight-ux.vercel.app"
  },
  "review": { "depth": "deep", "synthesis_mode": "disagreement" },
  "panel": {
    "personas": ["tom-random-visitor", "jamie-accessibility", "elena-founder-skeptical"],
    "risk_focus": ["empty-state-confusion", "trust-without-provenance", "screenshot-export-friction"]
  }
}

2. Normalized findings

After a BYOK run, the model output is converted into editable findings. These are the rows that become `run.json` and can be scored later.

[
  {
    "id": "finding-001",
    "persona": "tom-random-visitor",
    "issue_class": "empty-state-confusion",
    "severity": "moderate",
    "confidence": "medium",
    "raw_finding": "The first-time user may not know whether to build a brief, run a model, or export JSON first.",
    "evidence": "The surface exposes several artifact actions in one area.",
    "recommendation": "Keep the primary action focused on Build brief, then progressively reveal run and export actions."
  },
  {
    "id": "finding-002",
    "persona": "elena-founder-skeptical",
    "issue_class": "trust-without-provenance",
    "severity": "severe",
    "confidence": "medium",
    "raw_finding": "A skeptical user will not run BYOK unless the request path and storage posture are explicit.",
    "evidence": "API keys and screenshots are sensitive artifacts.",
    "recommendation": "Show request path, provider, model, prompt version, and screenshot count in every export."
  }
]

3. Repo-ready report

The final Markdown includes provenance, product surface, structured findings, raw panel output, and validation notes. It is designed to be checked into a product repo or attached to a PR.

# Preflight UX Report: Preflight UX web UI

## Provenance

- Provider/model: anthropic/claude-sonnet-4-6
- Prompt version: web-byok-v0.4
- Review depth: deep
- Synthesis mode: disagreement
- Screenshots: 2

## Structured Findings

### finding-001: `empty-state-confusion`

- Persona: `tom-random-visitor`
- Severity: `moderate`
- Confidence: `medium`
- Recommendation: Keep the primary action focused on Build brief.

## Validation Notes

Treat findings as pre-ship hypotheses until checked against user behavior.
