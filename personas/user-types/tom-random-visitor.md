# Tom — Random Visitor

## Profile

28, generalist knowledge worker. Landed from a friend's retweet. No
context. Has 60 seconds before he closes the tab.

## Expectations on first encounter

- "What does this do?"
- Wants to see it work in 30 seconds without committing to anything
- Will not read more than two sentences of explainer

## Drop-off vulnerabilities

- **no-empty-state-example** — sees a blank form, doesn't know what to
  paste → closes
- **explainer-only** — explainer text without a live example → closes
- **commitment-required-too-early** — must paste real content before
  seeing what the tool does → closes
- **value-not-legible-in-5-seconds** — has to imagine what the value would
  be → closes

## Behavioral patterns

- Reads headline
- Reads explainer
- If empty form: closes the tab
- If pre-loaded example response: reads it, understands the genre, may
  paste

## Sharing behavior

- Will retweet if the experience was crisp and the result was screenshot-
  worthy
- Will close silently if confused

## What Tom validates

- Empty state with live exemplar (single biggest conversion lift)
- Headline that promises a specific outcome
- Result format that telegraphs value before he commits

## What Tom breaks

- Empty form with no example
- Explainer-heavy hero
- Required commitment before any value is shown
- Slow first paint

## Calibration notes

v0.1: validated qualitatively against the initial applied product surface. Tom's
drop-off was the highest-leverage finding; drove the empty-state-exemplar
shipped change.
